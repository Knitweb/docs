# gither

Documentation for **Gither** — Knitweb's decentralized code forge: the ownership,
change-review, and release-gating layer above Git. GitHub/GitLab become optional
mirrors, not the authority.

## Contents

- **[overview.md](overview.md)** — what Gither is and how it relates to the Knitweb stack.
- **[launch-plan.md](launch-plan.md)** — prototype → launched product.
- **[commercial-strategy.md](commercial-strategy.md)** — go-to-market and positioning.
- **[decentralized-websites.pdf](decentralized-websites.pdf)** — white paper:
  *Decentralized Websites — Findings & the Role of Gither*. How to move dynamic sites
  (e.g. `sha2.fail`, `sha256.fail`) off centralized hosting onto crypto-paid P2P
  infrastructure (Akash for dynamic, IPFS/Arweave for static), funded self-custody via
  NMR → Uniswap → bridge → AKT/USDC, with a reversible DNS cutover — and where Gither
  fits as the forge layer that completes the picture.
  Source: [`decentralized-websites.html`](decentralized-websites.html) (kami long-doc).

See [`../REPOSITORIES.md`](../REPOSITORIES.md) for how Gither relates to the rest of the
Knitweb constellation (pulse, lens, vbank, monitor, virtualpc, bt).
